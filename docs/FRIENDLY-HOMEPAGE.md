# 친절한 홈페이지 공식 (Friendly Homepage Playbook)

> "사용자가 친절하다고 느끼는 홈페이지"의 특성을 유명 사이트 10곳에서 패턴화·공식화한 문서.
> 우리 Skill Finder에 바로 적용하기 위한 진단·처방 포함.

**작성일** 2026-07-22 · **방법** 6곳 실측(Stripe, Notion, Linear, Vercel, Product Hunt, Figma) + 4곳 정립된 패턴(Airbnb, Duolingo, Apple, Spotify)

---

## 0. 한 줄 정의

> **친절함이란 "생각을 시키지 않는 것"이다.** 사용자가 *이게 뭐지? 나한테 맞나? 뭘 누르지?* 를 고민하지 않아도 되게 만드는 것.

**공식:**

```
친절함 = (명확함 × 안내 × 따뜻함 × 낮은 마찰) ÷ 인지부하
```

- 분자를 키우고(명확·안내·따뜻·마찰↓), **분모(인지부하)를 줄인다.**
- 3초 테스트: 들어와서 **3초 안에** "무엇 / 누구를 위한 것 / 뭘 하면 되는지"가 보이면 친절, 아니면 불친절.

---

## 1. 10개 사이트 분석

| 사이트 | 핵심 한 줄(혜택) | 주 행동(CTA) | 용도/카테고리 노출 방식 | 친절 포인트 |
|---|---|---|---|---|
| **Stripe** | "매출을 키우는 금융 인프라" | Get started (무료·즉시) | 섹션 헤딩 + 고객 사례 카드 | 점진적 공개, 증거→요청 순서 |
| **Notion** | "팀과 AI가 함께 생각하는 곳" | Get Notion free | **역할·팀규모별 아이콘 카드**("Eng, Design, Marketing / Startup, Enterprise") | 결과 중심 카피, 무료 우선 |
| **Linear** | "제품 개발 시스템" | Get started | **워크플로 단계별**(Intake→Plan→Build→…) + 실제 스크린샷 | 기능 나열 대신 워크플로, 여백 |
| **Vercel** | "Develop. Preview. Ship." | Deploy now | 고객 사례 + 기능 콜아웃 | 3단어 명확성, 즉시 배포 |
| **Product Hunt** | "테크 신제품 발견" | 오늘의 랭킹 브라우징 | **제품마다 태그 칩 + 푸터 카테고리 그리드 + 트렌딩 사이드바** | 다중 발견 경로, 일관 카드, 투표=사회적 증거 |
| **Figma** | "하나의 워크스페이스" | Get started for free | **역할·용도·산업별 카드 + 미리보기 이미지** | 시각 미리보기로 "뭘 할 수 있나" 즉시 |
| **Airbnb** | (검색이 곧 메시지) | 거대한 검색 1개 | **아이콘 카테고리 바**(가로 스크롤, 시각적) | 주 행동 압도적 크게, 카테고리를 "보이게" |
| **Duolingo** | "무료로 언어 학습" | 큰 초록 버튼 1개 | 단계형 온보딩(선택지 최소) | 마스코트·색·격려 = 따뜻함, 선택 부담 0 |
| **Apple** | (제품 이미지가 메시지) | Learn more / Buy | 큰 시각 + 최소 텍스트 | 여백·집중·위계 |
| **Spotify** | "모두를 위한 음악" | 무료로 시작하기 | 감정적 가치 + 단일 CTA | 색·감정·단순함 |

## 2. 공통 패턴 → 10 원칙 (공식화)

### P1. 5초 명확성 (Clarity)
**한 문장으로 "혜택"을 말한다.** 기능("임베딩 하이브리드 검색")이 아니라 결과("필요한 걸 바로 찾아드려요").
→ 10곳 전부 혜택 중심 헤드라인. Stripe "매출을 키우는", Notion "함께 생각하는".

### P2. 단 하나의 주 행동 (One Primary Action)
지배적 CTA **1개**. 대부분 "무료로 시작". 선택을 강요하지 않는다.
→ Airbnb=검색, Duolingo=버튼 1개, Vercel=Deploy now.

### P3. 목적 단위 진입점을 "보이게" (Visible, Concrete Entry Points) ⭐
카테고리/용도/역할을 **드롭다운에 숨기지 말고** 아이콘·카드·칩으로 **화면에 펼친다.**
→ **Airbnb 아이콘 카테고리 바**, **Product Hunt 태그+푸터 그리드**, **Notion/Figma 역할·용도 카드**.
→ 이게 "브라우징이 쉽다=친절하다"의 핵심. **드롭다운 `<select>`는 카테고리를 숨기는 안티패턴.**

### P4. 일관된 스캔 패턴 (Scannable Consistency)
모든 항목이 **동일한 레이아웃**(이미지·제목·한줄·뱃지·지표). 눈이 패턴을 학습해 훑기 쉬움.
→ Product Hunt: 21개 항목이 전부 같은 구조.

### P5. 여백과 위계 (Whitespace & Hierarchy)
빽빽함은 불친절. **넉넉한 여백 + 크기 대비**로 눈을 이끈다.
→ Stripe·Notion·Linear 전부 "whitespace-heavy, approachable not overwhelming".

### P6. 사람의 말투 (Human Warmth)
"당신"에게 말하고, **결과 중심**, 전문용어 제거, 약간의 성격/격려.
→ Notion "완벽한 노트, 매번", Duolingo 마스코트·격려.

### P7. 신뢰 신호 (Social Proof)
숫자·로고·랭킹·"무료"로 불안을 없앤다. **요청(CTA)보다 증거를 먼저.**
→ Stripe "$1.9T 처리·99.999%", Figma "포춘500의 95%".

### P8. 점진적 공개 (Progressive Disclosure)
표면은 **단순**, 깊이는 **선택**. 한 번에 다 보여주지 않는다.
→ Stripe·Figma 명시적. 초보는 편하고 고수는 파고들 수 있게.

### P9. 구체 > 추상 (Show, Don't Tell)
실제 스크린샷·미리보기·예시로 "뭘 할 수 있나"를 즉시 보여준다.
→ Linear 실제 이슈 화면, Figma 미리보기 카드.

### P10. 살아있는 반응 (Alive & Responsive)
호버·마이크로인터랙션·부드러운 모션 → "관리되는 살아있는 제품" 느낌.
→ 과하지 않게(Linear "minimal motion").

## 3. 우리 Skill Finder 친절도 진단

| 원칙 | 현재 | 진단 |
|---|---|---|
| P1 명확성 | △ | 헤드라인은 있으나 **"스킬이 뭔지" 설명 0** — 초보는 맥락 없음 |
| P2 주 행동 | ○ | 큰 검색바 있음 (단, 라이브는 키워드라 자연어가 약함) |
| **P3 진입점 가시성** | **✗** | ⭐ **카테고리가 드롭다운(`<select`)에 숨어 안 보임.** 컬렉션 칩은 넣었지만 카테고리 자체가 묻힘 = **최대 불친절 지점** |
| P4 스캔 일관성 | ◎ | 행 레이아웃 일관 — 잘됨 |
| P5 여백·위계 | ○ | 애플풍, 준수 |
| P6 사람 말투 | △ | 다소 건조·기술적("수집/설명 미보강") |
| P7 신뢰 신호 | ◎ | 안전·신선도·검증·스타 뱃지 = **경쟁사보다 앞선 강점** |
| P8 점진적 공개 | △ | 전부 한 번에, 초보 가이드 없음 |
| P9 구체>추상 | △ | 텍스트 설명뿐, 미리보기·아이콘 약함 |
| P10 반응 | ○ | 호버·복사 피드백 있음 |

**한 줄 진단:** 정보 밀도·신뢰 신호는 강한데, **"어디로 가야 할지 보여주는 안내(P3)"와 "따뜻함(P6)·맥락(P1)"이 약해서** 불친절하게 느껴짐.

---

## 4. 처방 (우선순위)

1. **⭐ 카테고리를 보이게 (P3)** — 드롭다운 → **아이콘 카테고리 바/그리드**(Airbnb·Product Hunt식). 상위 카테고리(개발·문서·디자인·보안·데이터…)를 홈에 아이콘 칩으로 펼치기. 컬렉션 칩과 시각적으로 구분(컬렉션=목적, 카테고리=분야).
2. **온보딩 한 줄 + 뱃지 범례 (P1·P8)** — "Claude 스킬 = 클로드에게 특정 작업을 가르치는 지시셋. 골라서 `설치 복사` 한 번." + 검증/수집/안전/주의/위험 뱃지 뜻 1줄.
3. **말투 따뜻하게 (P6)** — "필요한 스킬, 용도만 말하면 찾아드려요" 톤. "설명 미보강" → "설명 준비 중" 같이 부드럽게.
4. **구체성 (P9)** — (후순위) 스킬 미리보기/설치 미리보기.

**핵심:** P3(카테고리 가시성)이 사용자가 지적한 바로 그 문제이자, 10개 사이트가 공통으로 잘하는 지점. **여기부터 고치면 체감 친절도가 가장 크게 오른다.**
